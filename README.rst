=====================
E.S.R.Labs Code Style
=====================
Code style is a matter of personal taste and discussions about it tend to become
emotional. In order to prevent religious wars, we define an E.S.R.Labs set of
style rules.

Rule 1: Tabs vs. Spaces
=======================
All of our code should be **tab-free**! For *indentation* use **4 spaces.**

Example
-------

======================================  ======================================
Good                                    Bad
======================================  ======================================
.. literalinclude:: ex_g_1.cpp          .. literalinclude:: ex_b_1.cpp
======================================  ======================================

Summary
-------
:+1: Indentation 4 spaces

:-1: Tabs
